NoteMute

note.com の記事をユーザー名・キーワードでミュート（非表示）する拡張機能

v1.5.0
Chrome / Edge
Manifest V3
GitHub 一覧に戻る

note.com のタイムライン・検索・プロフィールページから、指定したユーザーやキーワードを含む記事を非表示にする Chrome 拡張機能です。有料記事・メンバーシップ記事のフィルタリングにも対応。

機能

ミュート

  • ユーザーミュート — 指定ユーザーの記事をすべて非表示
  • キーワードミュート — タイトル・本文にキーワードを含む記事を非表示（AND/OR 切替対応）
  • 除外キーワード — 特定キーワードを含む記事はミュートしない（ホワイトリスト）
  • ワンクリックミュート — 記事カード上のボタンからモーダルで即座にミュート
  • 有効期限 — 1h / 1d / 7d / 30d / 永久から選択可能
  • コンテキストメニュー — 右クリックからユーザーミュート・キーワードミュート

記事フィルタ

  • 有料記事フィルタ — バッジ付きの有料記事を自動非表示
  • メンバーシップ記事フィルタ — メンバーシップ限定記事を自動非表示

プリセット

  • AI記事フィルタ — ChatGPT / Claude / Gemini 等の AI 関連キーワードを一括ミュート
  • 炎上ワードフィルタ — 炎上・批判系キーワードを一括ミュート
  • スパムフィルタ — 副業・情報商材系キーワードを一括ミュート

その他

  • リアルタイム反映 — ミュート設定変更が全タブに即時反映
  • SPA 完全対応 — Navigation API / 無限スクロール・ページ遷移でも漏れなく検知
  • 統計ダッシュボード — 日別・月別・ユーザー別のミュート統計を表示
  • インポート / エクスポート — ミュート設定を JSON で共有・バックアップ
  • ダークモード対応 — OS / 手動でテーマ切替（自動・ライト・ダーク）
  • 完全クライアントサイド — サーバーへの通信なし、プライバシー安全

インストール

方法 1: ソースからビルド


        # 1. リポジトリをクローン
        git clone https://github.com/<your-username>/note-mute-browser.git
        cd note-mute-browser
        
        # 2. 依存関係をインストール
        npm install
        
        # 3. ビルド
        npm run build

方法 2: ビルド済みフォルダを使用

note_mute_browser/ フォルダがビルド済み成果物です。

Chrome にロード

1. Chrome で chrome://extensions/ を開く

2. 右上の 「デベロッパーモード」 を ON にする

3. 「パッケージ化されていない拡張機能を読み込む」 をクリック

4. note_mute_browser/ フォルダ（ビルド出力）を選択

5. NoteMute のアイコンがツールバーに表示される

Edge / Brave にロード

同じ手順です。Edge は edge://extensions/、Brave は brave://extensions/ を使用してください。

使い方

ポップアップからミュート

1. ツールバーの NoteMute アイコンをクリック

2. 「ユーザー」または「キーワード」タブを選択

3. 入力欄にユーザー名またはキーワードを入力して追加

記事カードからミュート

1. note.com の記事カードにマウスを乗せる

2. 右上に表示されるミュートボタンをクリック

3. モーダルでユーザーミュートまたはキーワードミュートを選択

4. 有効期限を選んで確定

コンテキストメニュー

1. note.com 上でユーザーリンクを右クリック → 「このユーザーをミュート」

2. テキストを選択して右クリック → 「"..." をキーワードミュート」

有料記事フィルタ

1. ポップアップの設定パネルを開く

2. 「有料記事を非表示」にチェック

3. タイムライン上のバッジ付き記事が自動的に非表示に

グローバル ON/OFF

ポップアップ上部のトグルスイッチで、拡張機能全体の有効/無効を切り替えられます。

パフォーマンス最適化

  • MutationObserver バッチ処理 — 250ms デバウンスで DOM 変更をバッチ処理
  • WeakSet / WeakMap キャッシュ — 処理済みカード・ユーザー名をメモリ効率よくキャッシュ
  • キーワード事前分割 — exclude / normal を事前分割し、カードごとの filter() を排除
  • toLowerCase 重複排除 — キーワードマッチ時に title/body の小文字変換を1回に集約
  • CSS 一元管理 — manifest CSS のみで隠蔽スタイルを管理（JS 注入を廃止）
  • リトライタイマー自動キャンセル — MutationObserver 発火時に不要な初期化リトライを打ち切り
  • Promise.all 並列化 — ミュートリスト・設定の取得を並列実行
  • pendingNodes Set — 重複ノードの自動排除
